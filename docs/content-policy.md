# Content Policy

Kambradu separates code licensing from language content licensing.

## Code

The application code is MIT licensed.

## Dictionary-Derived Content

The local file `kristang dictionary.pdf` is a private reference file and must not be committed to the public repository. Bulk extracted text from that PDF must also stay out of Git.

Curated public entries derived from the dictionary must include:

- Source label: `Baxter and de Silva Kristang dictionary`
- Authors: `Alan N. Baxter and Patrick de Silva`
- Publisher: `Pacific Linguistics, The Australian National University`
- Year note: `2005; online edition 2015`
- DOI/source note: `10.15144/PL-564.cover`
- License: `CC BY-SA 4.0`
- Access level and review status

## Community Contributions

Every user submission must include:

- Content type: word, phrase, story, or note
- Language/community
- Provenance
- Consent statement
- Attribution name
- Access level
- Review status

Submitted material is not public until a steward approves it. Stewards may approve, reject, request edits, restrict access, or archive material.

## Restricted Content

Restricted content can include private family recordings, sensitive cultural notes, or material with incomplete consent. Do not cache restricted content in the public service worker shell, expose it in public search, or include it in open exports.
