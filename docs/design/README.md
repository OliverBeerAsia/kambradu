# Kambradu Design Reference

The first prototype is based on `kambradu-dashboard-concept.png`.

The intended app surface is a real working dashboard, not a marketing page:

- Kristang-first learner/contributor home
- audio-first saved words and lexical preview
- quick recording and upload actions
- community stewardship, access status, and review queue
- visible PWA sync/offline state

## HeadsUp / LevelHeads Direction

Kambradu now uses the related HeadsUp and LevelHeads product system as its local MVP baseline:

- dark operational rail with compact grouped navigation
- yellow, black, white, and restrained support colors instead of a soft single-hue palette
- hard borders, low radii, and offset yellow shadows for active or primary states
- Montserrat display type for workspace headings and Inter for product chrome
- dense but readable dashboard surfaces rather than marketing-style page sections
- Hegel, the HeadsUp owl, as the app mark, rail guide, and PWA icon

The design should keep public browsing, learner tools, contributor flow, and steward review visually connected while making access, consent, source, and review state easy to scan.
